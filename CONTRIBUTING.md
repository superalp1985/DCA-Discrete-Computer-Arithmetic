# Contributing

Contributions are welcome if they keep the draft modest, checkable, and useful for finite computation.

## Preferred Contributions

- Corrections to definitions, proof sketches, and implementation boundaries.
- Better references to existing systems, libraries, papers, or formalizations.
- Examples that can be represented by finite data structures or executable algorithms.
- Translation fixes for the English or Chinese versions.

## Style

- Keep the 43-part structure unless there is a strong reason to discuss a structural change first.
- Avoid claims that DCA replaces classical mathematics.
- Distinguish definitions, proof sketches, implementation notes, and approximation layers.
- Cite primary or stable sources where possible.

## Pull Request Checklist

- The Markdown renders cleanly.
- Links are checked.
- New claims have references.
- The tone remains low-key and implementation-oriented.
